# 抖音评论定向筛选与分析工具

针对抖音评论区数据，实现对指定用户的骚扰内容检测、对话抽取与多维度统计分析，支持关键词字典匹配、大语言模型 (LLM) 双方法，以及自动回退机制。

## 功能清单

- **数据库连接**：从 MySQL 数据库只读读取评论表，支持交互式选择库、表。
- **用户评论提取**：根据 `user_id`、`sec_uid` 或 `short_user_id` 提取指定用户全部评论（含完整列），单独输出 Excel。
- **骚扰内容检测**：
  - **关键词字典**：从 `Input` 文件夹的 `.txt` 文件（空格分隔）加载敏感词，进行快速匹配。
  - **大语言模型**：兼容 OpenAI 接口（已适配 DeepSeek 等），支持开启/关闭思考模式，自动重试与答案解析。
  - **混合模式**：先关键词匹配，未命中部分再由大模型补充检测。
  - **智能回退**：若首选方法失败（如无字典、API 未配置），自动切换备用方法，全部失败则记录日志并清理过程文件。
- **对话定位**：根据 `comment_id` 与 `parent_comment_id` 关系，找出被目标用户回复的骚扰评论，生成父‑子对话 Excel。
- **统计报告**：计算骚扰评论占比、被回复骚扰评论占比、平均回复次数等多个指标，输出独立分析文件。
- **LLM 详情输出**：单独记录每条评论的 LLM 最终回答与推理过程（`reasoning_content`），便于调优。
- **健壮性保障**：
  - 错误日志按时间戳写入根目录 `ErrorLog_*.log`。
  - 程序中断时自动清理已生成的过程文件。
  - `--clearconfig` 命令行可一键删除 `config.json` 重新配置。

## 依赖环境

- Python 3.9+
- 依赖库（可使用 `pip` 或 `uv` 安装）：
  ```
  pymysql pandas openpyxl questionary requests openai
  ```

## 快速开始

1. **安装依赖**
   ```bash
   pip install pymysql pandas openpyxl questionary requests openai
   ```
   或使用 `uv`：
   ```bash
   uv add pymysql pandas openpyxl questionary requests openai
   ```

2. **运行程序**
   ```bash
   python main.py
   # 或
   uv run main.py
   ```

3. **首次运行配置**
   程序会检测是否存在 `config.json`，若不存在则进入交互式配置向导：
   - MySQL 连接信息（主机、端口、用户名、密码）
   - 输入文件夹（存放关键词 `.txt`）和输出文件夹
   - 默认检测方法（仅关键词 / 仅 LLM / 两者结合）
   - 大模型 API 配置（Base URL、API Key、模型选择、提示词、思考模式开关）
   - 输出文件冲突处理（覆盖或自动重命名）

   后续运行将通过配置文件自动加载，也可选择重新配置。

## 配置文件与命令行参数

- **配置文件**：`config.json` 位于项目根目录，所有可修改参数均在其中（包括模型、提示词、检测方法等）。
- **清除配置**：如需重置所有配置，运行：
  ```bash
  python main.py --clearconfig
  ```
  确认后配置文件被删除，程序退出，下次启动将重新引导配置。

## 输入文件夹说明

- 将关键词字典放入 `Input` 文件夹，文件格式为 `.txt`，内容以**空格、换行**等空白符分隔，例：
  ```
  处吗 谈吗 CPDD 约吗 宝宝 老婆 开你
  ```
- 若选择关键词检测或两者结合模式，程序会自动让您从 `Input` 中选择一个 `.txt` 文件作为字典。

## 输出文件

所有输出保存在 `Output` 文件夹（可配置），文件命名格式为 `{目标用户昵称}xxx.xlsx`：

| 文件名 | 内容 |
|--------|------|
| `{昵称}评论内容.xlsx` | 目标用户的所有评论（完整原始列） |
| `{昵称}骚扰检测详情.xlsx` | 排除目标用户后每条评论的 LLM 响应与推理过程（仅限 LLM 方法） |
| `{昵称}回复评论及原评论内容.xlsx` | 被目标用户回复的骚扰父评论 + 目标用户的子回复，添加“角色”列 |
| `{昵称}评论回复分析.xlsx` | 多项统计指标（骚扰评论占比、被回复占比、平均回复次数等）|

**覆盖/重命名**：可在配置中选择是否覆盖同名文件；若禁止覆盖，则自动添加 `-1`、`-2` … 后缀。

## 使用流程

1. 启动程序，自动加载或配置数据库连接。
2. 通过方向键选择数据库 → 选择表。
3. 输入目标用户 ID 类型（`user_id`/`sec_uid`/`short_user_id`）及具体值。
4. 程序自动：
   - 提取目标用户评论并输出。
   - 对非目标用户的非空评论进行骚扰检测（根据所选方法，可能调用 LLM）。
   - 生成骚扰检测详情、对话 Excel、统计报告。
5. 查看控制台摘要，或打开输出文件夹中的 Excel 分析结果。

## 模块结构（方便二次开发）

```
harasscommentanalyze/
├── main.py                # 主流程调度
├── config.py              # 配置管理与交互式向导
├── db_utils.py            # MySQL 连接与数据读取
├── file_utils.py          # 文件操作（选择、保存、日志、清理）
├── detection/             # 骚扰检测组件包
│   ├── __init__.py        # 统一入口，回退逻辑
│   ├── keyword.py         # 关键词检测类
│   └── llm.py             # 大模型检测类（支持思考模式）
├── Input/                 # 输入文件夹，存放关键词 .txt
├── Output/                # 输出文件夹，存放 Excel 结果
├── config.json            # 运行时配置
└── ErrorLog_*.log         # 错误日志（出错时生成）
```

如需更换不同的大模型，只需修改 `detection/llm.py` 中的 `LLMDetector` 类的 API 调用部分；  
如需更改骚扰判定规则，可直接修改 `config.py` 中的默认提示词，或运行时重新配置。

## 注意事项

- API Key 会被明文保存在 `config.json` 中，请妥善保管。
- LLM 检测需要网络通畅，若不希望依赖外部服务，请选择“仅关键词匹配”方法，并准备好关键词字典。
- 思考模式开启仅对支持该特性的模型有效（如 deepseek‑v4‑pro），关闭时程序性能更高。
- 数据库连接仅执行 **SELECT** 查询，不会修改任何数据。
